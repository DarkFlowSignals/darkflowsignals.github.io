  • Entry date + price: already in signals_premium.json as signal_date and entry_premium
  • Day N counter: compute from signal_date on render
  • DTE: compute from expiry on render
  • Sparkline: needs new per-signal time-series capture (add JSONL append in position_manager.py 5min cycle, ~2-3 hr to wire)
  • Greeks: alpaca_extras.get_option_chain_with_greeks() already exists — wire into the 5min cycle and write greeks field to signals_premium.json
  • Earnings (next + prev): next is in {today}_earnings_calendar.json; prev needs weekly yfinance cron writing earnings_history.json
  • Cohort: already in signals_premium.json per signal — just surface
  • Thorough thesis: needs schema upgrade. Currently cowork_fire.py --thesis takes one string. Change to accept --thesis-setup + --thesis-macro + --thesis-risk (or one structured arg). Backfill from Smart_Money_Trades.md per-candidate sections — those already have setup + macro + risk paragraphs in the operator's writeup
  • Watchlist trigger zone: currently in signal metadata as entry_trigger block (per signal_trigger_monitor.py). Add to JSON for watchlist-tier signals

My recommendation: Layout A (current % leading). Reasons: (1) it's the live decision-relevant number for a member checking their book, (2) peak still gets prominent secondary placement so signal-quality story isn't lost, (3) most trading apps lead with current — we don't need to fight the convention since peak is already differentiated by color treatment (green-soft) and label.
