Data status (real talk):
  • Sparklines: Need a per-signal time-series of premium % return. We currently store entry/peak/current scalars but no time-series. Easiest path: position_manager.py 5-min cycle already prices every open position, so add a signals/{ticker}_{strike}_{expiry}_history.jsonl append on each cycle. After 1 day of capture, sparklines have real data; backfill is harder (needs Alpaca historical option pricing per signal).
  • Next earnings: Already in distribution/{today}_earnings_calendar.json. Pulled daily by pre-flight. Map ticker → next earning date + AH/BMO + days-to in dashboard JS.
  • Previous earnings + beat/miss: Not currently captured. Easiest: yfinance .earnings_dates per ticker on a weekly cron, write to web/darkflow-landing/earnings_history.json. Beat/miss requires the EPS surprise, which yfinance also gives.
  • Thesis text: cowork_fire.py already takes --thesis at fire time. Currently it's printed but not propagated into signals_premium.json. One-line fix: append thesis field in the signal record on fire. Backfill: extract from signals/YYYY-MM-DD_Smart_Money_Trades.md per-candidate sections (the operator wrote thesis there during the mechanical era).
  • Cohort data: Already in signals_premium.json per signal (n, median_peak_pct, win_rate_peak_basis). Surfaced in all 3 variants.
